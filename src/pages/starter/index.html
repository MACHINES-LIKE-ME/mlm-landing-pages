Why this structure

The page is intentionally boring where it should be.

01

Local assets only

CSS, JavaScript, favicons, and manifests are committed locally and bundled by Vite.

02

Compact by default

HTML is minified after build, JS and CSS are minified by Vite, and size budgets fail CI.

03

Security reviewed

CSP metadata, referrer policy, unsafe DOM patterns, HTTP URLs, and secrets are checked.

Agent-safe process

Create, validate, commit, deploy.

The agent instructions in AGENTS.md make the expected workflow explicit for future contributors and coding agents. Every commit should pass the same command that CI runs.

  1. Add a page Create src/pages/<slug>/index.html and keep assets local.
  2. Register it Update src/landing-pages.json and the public index.
  3. Validate it Run npm run validate; the hook runs it before commits.
  4. Publish it Push to main; GitHub Actions deploys to Pages.
